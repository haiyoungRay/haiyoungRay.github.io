# 安装vue-cli 3.0

npm install -g @vue/cli  //-g 为全局安装 "vue-cli"
  • 默认你已经安装了nodejs

# 创建项目

# 创建vue-app项目：

vue create vue-project
  • 默认配置
  • 手动配置：babel ts 预编译 等等… 【选择这个】

# 手动配置

Check the features needed for your project: (Press <space> to select, <a> to t
oggle all, <i> to invert selection)
- Babel						//将ES6编译成ES5
- TypeScript				//JS超集，主要是类型检查
- Router					//vue路由
- Vuex						//vue状态管理
- CSS Pre-processors		//css预编译
- Linter / Formatter		//代码检查工具
- Unit Testing				//单元测试，开发过程中前端对代码进行自运行测试
- E2E Testing				//端对端测试
  • Unit Testing(站在程序员的角度测试)：测试是把代码看成是一个个的组件。从而实现每一个组件的单独测试，测试内容主要是组件内每一个函数的返回结果是不是和期望值一样。
  • E2E Testing(站在用户角度的测试)：端对端测试，属于黑盒测试，通过编写测试用例，自动化模拟用户操作，确保组件间通信正常，程序流数据传递如预期。

# 是否使用Class风格装饰器

  • 即原本是：home = new Vue()创建vue实例
  • 使用装饰器后：class home extends Vue{}

 Use Babel alongside TypeScript (required for modern mode, auto-detected polyfi
lls, transpiling JSX)? (Y/n)  y

# 路由模式

  • 路由使用历史模式? 这种模式充分利用 history.pushState API 来完成 URL 跳转而无须重新加载页面
Use history mode for router? (Requires proper server setup for index fallback 
in production) (Y/n) y
  • 参考文档：Vue Router(HTML5 History 模式)

# css预编译器

Pick a CSS pre-processor (PostCSS, Autoprefixer and CSS Modules are supported 
by default): (Use arrow keys)
> Sass/SCSS (with dart-sass)
- Sass/SCSS (with node-sass)
- Less
- Stylus
  • dart-sass是Sass的主要实现，有快速，易于安装等特点

# 代码格式化检测

Pick a linter / formatter config: (Use arrow keys)
> ESLint with error prevention only   		//只进行报错提醒
- ESLint + Airbnb config					//不严谨模式
- ESLint + Standard config					//正常模式
- ESLint + Prettier							//严格模式
- TSLint (deprecated) 						//typescript格式验证工具
